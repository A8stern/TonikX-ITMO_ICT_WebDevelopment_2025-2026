    Отчет по лабораторной работе 1

    Тема: Работа с сокетами

    Студент: Ковалев Глеб К3341 367291

    Задание 1. UDP-клиент и UDP-сервер (Hello)

    Цель и постановка задачи

    Реализовать обмен сообщениями между клиентом и сервером по UDP с использованием библиотеки socket: клиент отправляет «Hello, server», сервер отвечает «Hello, client».

    Используемые технологии

    • socket
    • UDP

    Краткое описание реализации

    Сервер создаёт UDP-сокет, привязывается к адресу/порту, принимает сообщение от клиента и отправляет ответ.
    Клиент создаёт UDP-сокет, отправляет сообщение серверу и принимает ответ.

    Запуск

    1) Запустить сервер:
    bash python3 task1_udp_server.py 2) Запустить клиент в отдельном терминале:
    bash python3 task1_udp_client.py

    Задание 2. TCP-клиент и TCP-сервер (Теорема Пифагора)

    Цель и постановка задачи

    Реализовать клиент-серверное TCP-приложение для выполнения математической операции (теорема Пифагора): клиент вводит параметры, сервер вычисляет результат и возвращает его.

    Используемые технологии

    • socket
    • TCP

    Краткое описание реализации

    Сервер принимает TCP-подключение, получает строку с параметрами катетов, валидирует значения и вычисляет гипотенузу. Результат отправляется клиенту.
    Клиент вводит катеты с клавиатуры, отправляет запрос серверу и выводит ответ.

    Запуск

    1) Запустить сервер:
    bash python3 task2_tcp_server.py 2) Запустить клиент в отдельном терминале:
    bash python3 task2_tcp_client.py

    Задание 3. HTTP-сервер (выдача index.html)

    Цель и постановка задачи

    Реализовать HTTP-сервер на socket, который возвращает HTML-страницу, подгружаемую из файла index.html.

    Используемые технологии

    • socket
    • HTTP поверх TCP

    Краткое описание реализации

    Сервер принимает HTTP-запрос, определяет путь и при обращении к “/” или “/index.html” читает index.html и возвращает его в HTTP-ответе. При неверном пути/ошибке возвращает 404/500.

    Запуск

    1) Подготовить файл страницы:
    - Убедиться, что index.html находится рядом с сервером: task3_html_server.py 2) Запустить сервер: ``` bash python3 task3_html_server.py ``` 3) Открыть в браузере: - URL:http://localhost:8080/`

    Задание 4. Многопользовательский чат (TCP + threading)

    Цель и постановка задачи

    Реализовать многопользовательский чат по TCP с использованием threading: сервер обслуживает несколько клиентов параллельно и рассылает сообщения всем подключенным пользователям.

    Используемые технологии

    • socket
    • threading
    • TCP

    Краткое описание реализации

    Сервер принимает подключения и создаёт отдельный поток на каждого клиента. Список клиентов хранится в общей структуре с блокировкой. Сообщения клиентов рассылаются всем участникам.
    Клиент подключается к серверу, отправляет ник и запускает поток для получения сообщений, параллельно позволяя вводить и отправлять сообщения.

    Запуск

    1) Запустить сервер:
    bash python3 task4_chat_server.py 2) Запустить 2+ клиентов в разных терминалах:
    bash python3 task4_chat_client.py

    Задание 5. HTTP-сервер (GET/POST: дисциплины и оценки)

    Цель и постановка задачи

    Реализовать веб-сервер на socket для обработки GET и POST: принимать дисциплину и оценку, сохранять данные в txt-файл и отдавать HTML-страницу со всеми оценками.

    Используемые технологии

    • socket
    • HTTP поверх TCP
    • urllib.parse (parse_qs)
    • Хранение данных: grades.txt

    Краткое описание реализации

    GET “/” возвращает HTML-страницу с таблицей оценок и формой добавления.
    POST “/add” принимает данные формы (subject, grade), проверяет корректность оценки (целое 2–5), добавляет в структуру и сохраняет в grades.txt в формате «Дисциплина|оценка|оценка». После сохранения возвращается обновлённая HTML-страница.

    Запуск

    1) Подготовить файл данных:
    - Убедиться, что grades.txt находится рядом с сервером и содержит начальные данные: task5_grades_server.py
    2) Запустить сервер:
    bash python3 task5_grades_server.py 3) Открыть страницу в браузере:
    - URL: http://localhost:8080/

    Итоговые выводы

    В ходе лабораторной работы были реализованы сетевые приложения на базе сокетов в Python: обмен по UDP и TCP, простой HTTP-сервер, многопользовательский чат с потоками, а также веб-сервер с обработкой GET/POST и хранением данных в текстовом файле.
